\begin{abstract}
La globalización es un fenónemo que está afectando a aspectos económicos, tecnológicos, sociales, políticos y culturales que se dan entre las personas, empresas y gobiernos de todo el mundo. Esto provoca que la industria y el comercio, entre otros, rompan con las fronteras nacionales y se expanda a todo el mundo con el objetivo de llegar a un mayor público y reducir costes.

La industria del software no es una excepción y también se está amoldando a las nuevas modas que se imponen con el paso del tiempo. Por ello, el software está pasando de ser desarrollado en una única ubicación a ser desarrollado en múltiples localizaciones dispersas por todo el mundo. A esto se le conoce como \textit{Desarrollo Global de Software}. Pero este paradigma también introduce una serie de problemas, como el control de la calidad del software, que necesitan de entornos y herramientas que ayuden con su gestión.

El sistema propuesto con este proyecto consiste en una herramienta que permitirá visualizar medidas e indicadores de calidad bajo el contexto del desarrollo global. Para ello será necesario sintetizar la gran cantidad de información con la que cuentan los profesionales e investigadores a cargo de este tipo de actividades y proporcionar mecanismos que les ayude a conocer  qué ocurre durante el ciclo de vida del software. Gracias a ello les será posible predecir qué riesgos pueden darse y tomar las mejores decisiones para evitarlos o paliar sus efectos.
\end{abstract}
