\section{Método y fases de trabajo}

Dada la naturaleza del proyecto, en el que se irán definiendo requisitos y funcionalidades a lo largo de todo su ciclo de vida, se ha optado por utilizar una metodología de desarrollo de software que permita adaptarse a este caso de estudio. Por estas razones, se ha seleccionado el \textbf{Proceso Unificado de Desarrollo} (en adelante PUD) como metodología de trabajo.

El PUD \cite{jac00} es una evolución del Proceso Unificado de Rational, que define un \textit{``conjunto de actividades necesarias para transformar los requisitos de usuario en un sistema software''}. Es un marco genérico que puede especializarse para una gran variedad de sistemas de software cualesquiera que sean el área de aplicación o el tamaño del proyecto \cite{pud-espinoza}. Sus caracteriza debido a que está dirigido por casos de uso, centrado en la arquitectura, y es iterativo e incremental (Fig. \ref{fig:mapa-conceptual-pud}).
