\section{Objetivos}

A continuación, y bajo el subtítulo de ``objetivo principal'', se describirá la meta final que se persigue en la elaboración de este Proyecto Fin de Carrera (en adelante, PFC), que consiste en obtener un producto software. Un producto software no se constituye únicamente de código fuente ejecutable, si no que también se compone de la documentación, diagramas y modelos -entre otros- que se han generado en todas y cada una de las fases que conforman todo el proceso de desarrollo del software. Esto incluye análisis de requisitos, diseño, implementación, pruebas, despliegue, mantenimiento y soporte. Ello acarrea la adquisición de una serie de competencias y la compleción de un conjunto de tareas que serán expuestas bajo el título de ``objetivos secundarios''.


\subsection{Objetivo principal} \label{sec:objetivo-principal}

El objetivo principal del PFC consiste en elaborar una \textbf{aplicación web} que permitirá la visualización de indicadores y métricas utilizadas en el contexto empresarial bajo el paradigma del \textbf{Desarrollo Global de Software}, escenificadas por un conjunto de metáforas previamente definidas. Esta visualización se llevará a cabo mediante tecnologías basadas en gráficos 2D y 3D, utilizándose en cada momento la que se considere oportuna en función de las necesidades de representación de la métrica deseada.

A fin de desarrollar un sistema extensible y que satisfaga los requisitos de visualización más exigentes, se desarrollará un \textbf{motor gráfico 3D} basado en \textit{Java} y \textit{OpenGL}, que será integrado en la aplicación Web mediante un applet. Asimismo, el usuario deberá ser capaz de interactuar con dicho applet por medio de un navegador Web convencional.

El motor gráfico implementará un conjunto de \textbf{metáforas de visualización} y permitirá que sean utilizadas para representar las distintas dimensiones de los datos o información que se quiera representar visualmente. Por otro lado, se dejarán abiertas determinadas guías para permitir la incorporación de nuevos modelos y metáforas de un modo fácil, viable, rápido y poco costoso por parte del desarrollador.

La visualización se podrá hacer en base a dos niveles de abstracción:
\begin{myitemize}
	\item \textbf{Nivel de fábrica}, en el que se podrá acceder a información relativa a una fábrica determinada o a un conjunto de fábricas. Una fábrica puede estar desarrollando varios proyectos.
	\item \textbf{Nivel de proyecto}, desde el que se podrá visualizar la información de un proyecto. Un proyecto puede estar desarrollándose en varias fábricas.
\end{myitemize}

A medida que los usuarios hagan uso del sistema, la cantidad de información procesada puede ser muy abundante. Por esta razón, será necesario representarla en distintos niveles de detalle. Estos niveles se denominan operativo, estratégico y táctico. La finalidad que se persigue es permitir al usuario la capacidad de realizar un análisis eficiente e intuitivo de las métricas e indicadores en cuestión, ya sea en un momento concreto del ciclo de vida de desarrollo o mediante históricos y/o tendencias.

La aplicación web será desarrollada utilizando tecnologías actuales empleadas en desarrollo y diseño web (Ver Sección \ref{sec:medios}) con el fin de obtener una RIA (Rich Internet Application) \cite{def:ria}. Deberá gozar de una alta usabilidad y navegabilidad, tratando de maximizar la experiencia de usuario.

Una RIA se fundamenta en una arquitectura cliente/servidor asíncrona, segura y escalable, disminuyendo la carga de trabajo en el servidor y siendo accesible desde cualquier navegador web. Así, estará disponible para cualquier persona desde cualquier rincón del planeta, siempre y cuando posea una conexión a Internet, tratando de maximizar la experiencia de usuario y producir un incremento en la productividad.


\subsection{Objetivos secundarios} \label{sec:objetivos-secundarios}

La consecución del objetivo principal acarrea la adquisición y compleción de las siguientes competencias y tareas:
\begin{myitemize}
	\item Revisión sistemática de las distintas técnicas de visualización empleadas para escenificar indicadores y métricas que representan información.
	\item Análisis del mundo real, en el contexto del GSD, para lograr un diseño conceptual que abstraiga la información relevante.
	\item Revisión sistemática de librerías para la generación de gráficos 3D y técnicas de visualización de datos empleando gráficos 3D.
	\item Definición de una arquitectura estable y extensible de un motor gráfico que permita la representación de modelos para escenificar metáforas de visualización, empleando buenas prácticas de programación así como patrones de diseño.
	\item Desarrollo de un motor gráfico para la visualización de datos empleando modelos en 3D.
	\item Revisión sistemática de tecnologías y \textbf{frameworks} para desarrollo de aplicaciones Web y tecnologías para la generación de gráficos y modelos de visualización en la Web.
	\item Definición de una arquitectura estable y extensible de una aplicación Web, utilizando patrones de diseño.
	\item Diseño de una base de datos para la gestión de la lógica de negocio utilizada por el sistema a implementar.
	\item Desarrollo de una aplicación Web para la visualización de información empleando técnicas basadas en gráficos en tres dimensiones.
	\item Integración del motor gráfico basado en OpenGL con la aplicación Web.
	\item Desarrollo de pruebas unitarias para el software.
	\item Elaboración de la documentación técnica asociada al proyecto.
\end{myitemize}
